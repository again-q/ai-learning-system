# 知识图谱结构规范

> 本文档定义知识图谱（K 维知识地图）的数据结构规范。
> 知识图谱 = 五维模型中的 K 维知识地图，描述"知识世界"和"学生在知识世界的位置"。
> 不负责存储 A/T/Q/S。

---

## 整体关系

```
教材/课程标准
        ↓
知识树（固定地图）
        ↓
学生知识状态（动态掌握度）
        ↓
五维诊断模型
        ↓
AI教练个性化辅导
```

---

## 一、知识图谱结构

核心：**树状主结构 + 两种辅助关系**，不做复杂知识网络。

```
学科
 └── 模块
      └── 章节
           └── 知识单元
                └── 子知识单元
```

树负责：
- 学生浏览
- K 掌握度聚合
- 学习路径规划

## 二、知识节点 JSON 规范

```json
{
  "knowledge_id": "math_function_domain_001",

  "basic": {
    "name": "函数定义域",
    "subject": "数学",
    "stage": "高中",
    "grade": "高一",
    "level": "knowledge"
  },

  "tree": {
    "parent_id": "function_001",
    "children_ids": [
      "radical_domain_001",
      "fraction_domain_001"
    ],
    "path": [
      "数学",
      "函数",
      "函数概念",
      "函数定义域"
    ]
  },

  "concept": {
    "summary": "函数中自变量允许取值范围",
    "keywords": [
      "自变量",
      "限制条件",
      "取值范围"
    ]
  },

  "relations": {
    "reference": ["inequality_001"],
    "related": ["function_graph_001"]
  },

  "importance": {
    "curriculum_weight": 4,
    "exam_frequency": 5
  }
}
```

---

## 三、字段说明

### 1. basic（身份）

回答：这是哪个知识？

| 字段 | 说明 | 取值示例 |
|------|------|---------|
| name | 知识名称 | "函数定义域" |
| subject | 学科 | "数学" |
| stage | 学段 | "高中" |
| grade | 年级 | "高一" |
| level | 层级 | "knowledge" |

不参与诊断，仅用于定位。

### 2. tree（核心）

回答：它在知识地图哪里？

| 字段 | 说明 |
|------|------|
| parent_id | 父节点 ID |
| children_ids | 子节点 ID 列表 |
| path | 从根到该节点的完整路径数组 |

为什么 tree 是第一公民：
- **学生看**：渲染目录导航，不用递归查 parent
- **系统算**：子节点掌握度 → 父节点掌握度聚合

### 3. concept（语义）

回答：AI 怎么理解这个知识？

| 字段 | 说明 |
|------|------|
| summary | 核心定义（一句话，不写长篇教材） |
| keywords | 关键词列表 |

### 4. relations（辅助道路）

| 关系 | 含义 | 示例 | 用途 |
|------|------|------|------|
| reference | 前置依赖 | 函数定义域 → 引用: 不等式、集合 | 发现基础缺陷 |
| related | 横向联系 | 函数定义域 → 相关: 函数图像、函数性质 | 探索和迁移 |

注意：不包含"包含"关系——树已经表达包含，重复维护增加生成难度。

### 5. importance（重要程度）

| 字段 | 说明 | 范围 |
|------|------|------|
| curriculum_weight | 课标权重 | 1~5 |
| exam_frequency | 考频 | 1~5 |

重要程度 ≠ 学生掌握度（掌握度单独存储）。

## 四、学生知识状态（独立存储）

知识节点不能存掌握度——同一个知识不同学生掌握度不同。

```json
{
  "student_id": "001",
  "knowledge_id": "math_function_domain_001",
  "mastery": 0.82,
  "evidence_count": 20,
  "last_update": "2026-07-17"
}
```

---

## 五、各维度与知识图谱的关系

| 维度 | 关系 |
|------|------|
| K | 绑定知识地图。输入：知识节点 + 学习证据 → 输出：每个知识点掌握度 |
| A | 独立计算。来源：解题步骤、操作过程。不修改知识图谱 |
| T | 全局能力。来源：跨知识连接、新情境应用。不是单点属性 |
| Q | 来源：思考过程、分析路径 |
| S | 来源：时间序列表现 |

---

## 六、AI 教练读取

AI 读取：学生知识状态 + 相关知识节点 + 五维报告。

不直接读教材。

---

## 七、知识点层级说明

| 层级 | 内容 | 计入 K 总分 | 生成方式 |
|------|------|-----------|---------|
| 知识层 | 概念定义、公式、性质 | ✅ 是 | AI 生成 |
| 模型层 | 题目模型、解题框架 | ❌ 参考输出 | 人工维护 |
| 方法层 | 思维技巧、解题策略 | ❌ 参考输出 | 人工维护 |

---

## 八、自动生成约束（给生成 AI 的 prompt 要点）

生成 AI 的任务不是"总结教材"，而是"生成可用于 K 维诊断的知识地图"。

约束：
1. 一个节点必须能独立掌握/遗忘
2. 不拆成公式碎片
3. 不合并多个认知过程
4. 保留树结构
5. 生成引用和相关关系
6. 输出固定 JSON
