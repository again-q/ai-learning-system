# 编码规范模板

使用本模板生成 `doc/detailed/编码规范.md`。
所有内容必须基于架构文档中的实际技术栈，不生成与项目无关的通用规范。

> 第3节"目标语言编码规则"的内容依 LC-001 语言决定，生成时只保留对应语言内容，删除其他语言内容。

## 2. 语言契约（Language Contract）

> gatekeeper 和 code-reviewer 在执行任何操作前，必须先读取本节，确保所有输出与此契约一致。

| 契约编号 | 契约项 | 值 | 说明 |
|----------|--------|-----|------|
| LC-001 | 目标语言 | {Java \| Python \| Go \| Node.js} | 所有代码实现必须使用此语言 |
| LC-002 | 语言版本 | {版本号} | 禁止使用高于此版本的专属特性 |
| LC-003 | Web 框架 | {框架名} | 路由、中间件、依赖注入均使用此框架 |
| LC-004 | 数据访问层 | {ORM/框架名} | 禁止混用 |
| LC-005 | 包管理工具 | {工具名} | 依赖声明和构建均使用此工具 |
| LC-006 | 测试框架 | {框架名} | 单元/集成测试均使用此框架 |
| LC-007 | 日志框架 | {框架名} | 禁止使用 System.out/print |
| LC-008 | 工具库偏好 | {工具库名} | 优先使用此工具库 |
| LC-009 | 构建产物 | {JAR/Package/Binary/Bundle} | 部署单元的形态 |
| LC-010 | 代码分层模型 | {各层名称} | 各层职责划分，禁止跨层调用 |
