# AI 学习系统 · 前端视觉与动画工程经验

> 本文档面向所有 AI 编码助手（TRAE / Reasonix / Cursor / Copilot 等）。
> 目标：让任何 AI 助手按此规范执行，都能产出与本项目一致的视觉与交互质量。
> 版本：v1.0 · 2026-07-02

---

## 一、核心设计哲学

**三个不可妥协的原则**：

1. **简约优先** — 不堆砌装饰。每个视觉元素必须有明确职责，删除一切「为了好看」的光晕、噪点、引号、装饰性图标。
2. **动画服务于交互** — 动画不是表演，是反馈。入场动画告诉用户「内容来了」，微交互告诉用户「你点中了」，加载态告诉用户「正在处理」。
3. **统一胜过个性** — 全 App 共用一套设计 Token、一套动画曲线、一套组件样式。不允许某个页面「特立独行」。

**反模式清单（禁止出现）**：
- ❌ 深色背景 + 鲜艳光晕（除非用户明确要求深色主题）
- ❌ 装饰性引号、装饰性分割线、装饰性 emoji 漫天飞
- ❌ 入场动画超过 1 秒（用户会觉得「卡」）
- ❌ 每个页面用不同的主色或圆角
- ❌ 按压反馈只有透明度变化（太弱，用户感知不到）

## 二、设计 Token（必须全 App 共用）

所有页面 wxss 顶部必须声明以下 CSS 变量，**数值不得擅自修改**：

```css
page {
  /* 背景层 */
  --bg:            #ffffff;   /* 主背景 */
  --bg-subtle:     #f5f5f7;   /* 分区/输入框/卡片背景 */

  /* 文字层（三级灰阶） */
  --text-primary:   #1d1d1f;  /* 标题、正文 */
  --text-secondary: #6e6e73;  /* 副标题、说明 */
  --text-tertiary:  #a1a1a6;  /* 占位符、次要信息 */

  /* 主色（iOS 蓝） */
  --accent:        #007aff;
  --accent-soft:   #4a9eff;
  --accent-bg:     rgba(0,122,255,0.06);

  /* 边框 */
  --border:        #d2d2d7;
  --border-soft:   #e5e5ea;

  /* 阴影（三级，主色阴影带光晕） */
  --shadow-sm: 0 1rpx 2rpx rgba(0,0,0,0.04);
  --shadow-md: 0 4rpx 16rpx rgba(0,0,0,0.06);
  --shadow-lg: 0 8rpx 32rpx rgba(0,122,255,0.18);

  /* 缓动曲线 */
  --ease-out:    cubic-bezier(0.16, 1, 0.3, 1);   /* 通用 */
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1); /* 弹性回弹 */
}
```

**使用规则**：
- 颜色、阴影、圆角必须用变量，禁止硬编码 `#007aff`
- 主色 `--accent` 仅用于：主按钮、可点击元素聚焦态、强调数值、链接
- 不可用主色做大面积背景（会刺眼）

## 四、动画规范（核心章节）

### 4.1 入场动画（Staggered Reveal）

**用途**：页面或区块首次显示时，元素依次淡入上移。

**标准参数**：
```css
@keyframes reveal-up {
  from { opacity: 0; transform: translateY(16-20rpx); }
  to   { opacity: 1; transform: translateY(0); }
}

.reveal    { animation: reveal-up 0.4s var(--ease-out) both; }
.reveal-1  { animation-delay: 0.04s; }
.reveal-2  { animation-delay: 0.10s; }
.reveal-3  { animation-delay: 0.16s; }
.reveal-4  { animation-delay: 0.22s; }
.reveal-5  { animation-delay: 0.28s; }
```

**关键约束**：
- ⚠️ **总时长不超过 0.8s**（最后一个元素完成时间 = 动画时长 + 最大 delay）
- ⚠️ **位移幅度 16-20rpx**，不要超过 24rpx（幅度大+时间长=卡顿感）
- ⚠️ **delay 间隔 60ms**，不要 100ms+（间隔大=等动画感）
- ⚠️ **5-6 级足够**，不要 8 级以上（用户没耐心看完）

**反例（禁止）**：
```css
/* ❌ 太慢，1.28s 才完成 */
.reveal { animation: reveal-up 0.7s var(--ease-out) both; }
.reveal-6 { animation-delay: 0.58s; }
```

### 4.2 页面切换淡入

**用途**：tab 切换时消除「硬切」感。

**实现模式（JS + CSS 配合）**：

```js
// page.js
data: { pageReady: false },

onShow() {
  this.setData({ pageReady: false });
  setTimeout(() => this.setData({ pageReady: true }), 16);
}
```

```xml
<!-- page.wxml -->
<view class="container {{pageReady ? 'page-enter' : 'page-leaving'}}">
```

```css
/* page.wxss */
.container {
  transition: opacity 0.3s var(--ease-out), transform 0.3s var(--ease-out);
}
.page-leaving { opacity: 0; transform: translateY(12rpx); }
.page-enter   { opacity: 1; transform: translateY(0); }
```

**关键约束**：
- ⚠️ **时长 0.3s**，不要 0.5s+（切换频繁，太慢拖沓）
- ⚠️ **位移 12rpx**，比入场动画幅度小（切换是轻量过渡）
- ⚠️ **Canvas 原生组件不响应 opacity 过渡**，需在 `pageReady:true` 后手动重绘，否则残留上一帧

### 4.3 微交互反馈

每个可点击元素必须有 `:active` 反馈，**不得只有透明度变化**。

| 元素类型 | 推荐反馈 |
|---------|---------|
| 主按钮 | `transform: scale(0.98)` + `opacity: 0.92` + 阴影变浅 |
| 图标块 | `transform: scale(0.92)` + 弹性回弹（`ease-spring`） |
| 列表项 | `background: var(--bg-subtle)` |
| 卡片 | `transform: scale(0.98)` |
| 输入框聚焦 | 描边/下划线从中心扩展到 100% + 变蓝 |

**标准按压样式**：
```css
.btn:active {
  transform: scale(0.98);
  opacity: 0.92;
  box-shadow: 0 4rpx 16rpx rgba(0,122,255,0.12);
  transition: all 0.25s var(--ease-out);
}
```

### 4.4 加载态

**双层反馈**（按钮内嵌 + 全局弹窗）：

```js
// 按钮内嵌 spinner
this.setData({ isLoading: true });
// 全局遮罩防双击
wx.showLoading({ title: '登录中', mask: true });

// 成功/失败都要 hide
wx.hideLoading();
```

**按钮 loading 态样式**：
- 背景色变浅（`--accent-soft`）
- 显示旋转 spinner
- 文案变为「XX中」
- `pointer-events: none` 防重复点击

### 4.5 AI 对话页特有动画

| 场景 | 动画 |
|------|------|
| 新消息入场 | `slideUp` 0.3s（opacity + translateY 16rpx + scale 0.96→1） |
| AI 正在输入 | 三点呼吸（`pulse` 1.2s 循环，delay 0/0.2/0.4s） |
| 输入框聚焦 | 背景变白 + 蓝色描边 `box-shadow: 0 0 0 2rpx rgba(0,122,255,0.2)` |

## 六、内容不可变原则

**视觉优化时，以下内容严禁修改**：

- ❌ 数据绑定（`{{xxx}}`）
- ❌ 事件绑定（`bindtap` / `bindinput`）
- ❌ 业务逻辑（JS 函数）
- ❌ 页面路由配置
- ❌ 云函数调用

**只能修改**：
- ✅ WXSS 样式
- ✅ WXML 的 class 名和结构（但不改绑定）
- ✅ 为动画新增的 data 字段（如 `pageReady`、`isLoading`）

## 七、多 AI 助手协作规范

### 7.1 分工原则

| AI 助手类型 | 适合分配的任务 |
|-------------|---------------|
| **桌面端编码助手**（如 Reasonix） | 后端云函数、CRUD 模板、新页面骨架、批量代码生成 |
| **IDE 内助手**（如 TRAE） | 前端视觉、动画、算法实现、架构梳理、代码 review、文档 |

### 7.2 协作契约

1. **设计语言规范是单一事实来源** — 所有 AI 助手必须遵循本文档
2. **前端代码产出后必须过视觉审查** — 骨架代码先生成，视觉打磨由专人助手处理
3. **JS 逻辑和视觉分离** — 一个助手改 JS，另一个助手改 WXSS，避免冲突
4. **变更同步** — 任何设计 Token 调整，必须同步到本文档并通知所有协作者

### 7.3 给 AI 助手的 Prompt 模板

```
请按照《前端视觉与动画工程经验》v1.0 执行：
1. 使用文档第二章的设计 Token，禁止硬编码颜色
2. 入场动画遵循 4.1 节参数（总时长≤0.8s）
3. 所有可点击元素必须有 :active 反馈（4.3 节）
4. 只改 WXSS 和必要的 WXML class，不动 JS 逻辑和 data 绑定
5. 视觉风格：简约白色，不堆砌装饰
```

## 八、性能与稳定性

### 8.1 动画性能

- ⚠️ **只用 transform 和 opacity** 做动画，不用 width/height/top/left
- ⚠️ **动画时长 ≤ 0.4s**（入场） / ≤ 0.3s（切换） / ≤ 0.25s（微交互）
- ⚠️ **避免同时动画元素超过 6 个**（小程序渲染能力有限）

### 8.2 Canvas 注意事项

- Canvas 2D 是原生组件，**不响应父级 CSS opacity/transform 过渡**
- 页面切换淡入后，必须手动调用重绘函数
- 绘制时使用 `dpr` 缩放避免模糊

```js
const dpr = wx.getSystemInfoSync().pixelRatio;
canvas.width = 320 * dpr;
canvas.height = 320 * dpr;
ctx.scale(dpr, dpr);
```

### 8.3 安全区适配

所有页面必须适配刘海屏：

```css
padding-top: calc(88rpx + env(safe-area-inset-top));
padding-bottom: calc(48rpx + env(safe-area-inset-bottom));
```

## 附录：关键经验教训

### 教训 1：过度装饰是 AI 生成代码的通病

**现象**：第一次视觉重做时加入了光晕、噪点、引号装饰、光泽流动，用户反馈「太有质感，没必要」。

**根因**：AI 助手倾向于「加法」思维，认为更多视觉效果=更好。

**对策**：明确「简约优先」原则，装饰元素必须有明确职责，删除一切「为了好看」的元素。

### 教训 2：动画太慢会被感知为「卡」

**现象**：初版入场动画 0.7s + 0.58s delay = 1.28s，用户反馈「加载有点慢」。

**根因**：AI 助手容易把动画时长设得偏长（追求「流畅感」），但实际用户感知是「等待」。

**对策**：硬性约束总时长 ≤ 0.8s，位移 ≤ 20rpx，delay 间隔 ≤ 60ms。

### 教训 3：Canvas 不跟随 CSS 过渡

**现象**：页面切换淡入时，Canvas 环形图残留上一帧。

**根因**：小程序 Canvas 2D 是原生组件，渲染层级独立，opacity/transform 对它无效。

**对策**：onShow 触发淡入后，手动调用 `drawRing()` 重绘 Canvas。

### 教训 4：按压反馈不能只有透明度

**现象**：早期按钮 `:active { opacity: 0.85 }`，用户感知不到点击。

**根因**：纯透明度变化在快速点击时几乎不可见。

**对策**：按压反馈必须包含 `transform: scale()` 或背景色变化，透明度仅作为辅助。

### 教训 5：内容不可变是底线

**现象**：视觉优化时如果改动 JS 逻辑或数据绑定，容易引入 bug。

**根因**：视觉优化任务与功能开发任务边界模糊。

**对策**：明确「只改 WXSS 和 WXML class，不动 JS」的红线，新增 data 字段（如 pageReady）是允许的例外。
