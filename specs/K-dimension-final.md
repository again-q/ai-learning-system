# K 维度工程规格（定稿）

## 1. LLM 难度判定指令

判断题目难度时，从以下 5 个维度综合评估（每个维度 0.0-1.0）：

| 维度 | 含义 | 低分（0.1） | 高分（0.9） |
|------|------|------------|------------|
| 知识复杂度 | 涉及知识点数量、推导步骤 | 单步套公式 | 多步+跨知识点 |
| 思维深度 | 是否需要转换思路 | 直接套用 | 逆向/构造 |
| 陷阱密度 | 是否有隐蔽条件/易错点 | 直白无坑 | 多陷阱/分类讨论 |
| 计算强度 | 计算量大小 | 口算级别 | 复杂运算 |
| 陌生度 | 题型是否常见 | 常规题 | 完全没见过的新题 |

**计算方式**：以上五个维度加权平均（权重大致相同），输出 0.01-0.99 连续值。

---

## 2. 过程分判定

LLM 从学生作答判定过程质量：

| 分数 | 判定标准 |
|------|---------|
| 1.0 | 完全正确，思路清晰 |
| 0.5 | 部分正确 / 正确但思路模糊 |
| 0.3 | 错误但有正确思路 / 沾边 |
| 0 | 完全错误 / 空白 / 乱写 |

---

## 3. 掌握度计算

```
每道题贡献 = 难度(0.01-0.99) × 过程分(0/0.3/0.5/1.0)

知识点累计得分 S_k = Σ 贡献
知识点累计难度 D_k = Σ 难度

掌握度 = S_k / D_k
```

展示时分为 4 档（档位边界等有足够数据后按自然分布确定）：
- 🟢 掌握
- 🟡 巩固
- 🟠 薄弱（提分最高效）
- 🔴 紧急

---

## 4. 与旧方案对比

| 项目 | BKT 方案 | 加权得分方案 |
|------|---------|------------|
| 模型复杂度 | 高（4参数 + 遗忘曲线） | 低（无参数） |
| 数据需求 | 大量历史数据拟合 | 几道题即可 |
| 跨 batch 漂移 | 有 | 无（累计S_k/D_k） |
| 难度区分 | 无 | 有（0.01-0.99 连续） |
| 过程质量 | 无（只看对错） | 有（4档过程分） |
| LLM 依赖度 | 低 | 中（LLM判难度和过程） |
