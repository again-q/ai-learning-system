## 一、项目起源

重点高中普遍部署了校园学情管理系统，能动态追踪每个学生在每个知识点上的掌握情况。但这些系统的服务对象是**教师**——用于个性化出卷、学情分析、教学调整。学生本人依然处于信息黑箱之中：只知道分数，不知道「哪里不会」「为什么不会」「下一步该学什么」。

本项目的初衷，是把这种学情分析能力从教师侧搬到学生侧，让学生真正掌握自己的学习情况。

### 发起人背景

项目负责人现为初升高学生，深刻体会过课内学习中的真实痛点：题做了很多却说不清自己哪里会、哪里不会；考试失分后只能归因到「粗心」「没复习到」，无法定位到具体的知识盲区；想针对性补强却不知道从何下手。这些困惑的根源，是学生缺少一个能客观刻画自身学习情况的工具。

本项目正是基于这一亲历的痛点发起——以五维能力向量模型量化每个知识点的真实掌握程度，以 AI 对话支持针对性深入提问，与课内学习形成相辅相成的闭环。

## 二、系统目标

### 定位

**课后辅助工具，而非替代课堂学习。**

帮助学生巩固、查漏、规划，不代替学校教学。与课内学习形成相辅相成的关系——课堂学新知，课后用本系统消化吸收、查漏补缺、针对性深入。

### 核心目标

| 目标 | 说明 |
|------|------|
| **量化真实掌握** | 用五维向量（K/A/T/Q/S）追踪每个知识点的真实掌握程度，而非简单统计做题数 |
| **动态学习路径** | 根据薄弱点 AI 自动调整学什么、什么时候复习 |
| **全科统一管理** | 一个系统管理所有学科的知识图谱和掌握度 |
| **认知分层闭环** | 从学到练到测到补，形成完整学习链路，解决学习生活中的遗漏点 |

### 价值主张

一条链路打通学生学习生活：**了解学习情况 → 定位不懂之处 → 深入提问突破 → 与课内学习呼应**。

## 四、已实现：五维能力分析（知识图谱页）

作为系统的首个能力模块，知识图谱页采用**五维能力向量模型**对学科能力进行精细刻画：

| 维度 | 符号 | 含义 |
|------|------|------|
| Knowledge | K | 知识储备（概念、公式、定理） |
| Application | A | 知识应用（标准题型解题） |
| Thinking | T | 思维能力（推理、归纳、迁移） |
| Problem Solving | Q | 问题解决（综合题、开放题） |
| Speed | S | 解题速度与准确率 |

向量 `M = (K, A, T, Q, S)` 独立评估各维度，避免「一个总分掩盖薄弱点」。通过 Canvas 自绘中心辐射图 + 镜头缩放交互，让能力分布一目了然。

> 详细理论见 [五维能力向量框架-理论文档.md](五维能力向量框架-理论文档.md)

## 五、知识库架构与产品哲学

> 本节为知识库架构与产品定位的最终决策，单一事实来源见 [知识库架构决策文档.md](知识库架构决策文档.md)。

### 1. 架构：课标锚定 + 个人补充

知识库分两层：

- **课标层（统一锚点）**：基于国家课标 + 教辅细化 + 真题反推构建，覆盖高考考点，作为五维评分的统一基准。学生不可修改。
- **个人层（个性化补充）**：学生在课标节点上挂载个人笔记、关联、细分，仅影响学习路径推荐，**不影响五维评分**。可一键重置回课标模板。

```
┌─────────────────────────────────┐
│   课标知识库（只读 · 统一锚点）   │
│  课标 + 教辅 + 真题反推 + AI 校验 │
└──────────────┬──────────────────┘
               │ 学生在节点上挂载补充
               ▼
┌─────────────────────────────────┐
│       个人补充层（私有）          │
│  笔记 / 关联 / 细分子节点         │
│  仅影响学习路径，不参与五维评分    │
└─────────────────────────────────┘
```

**为什么不采用「学生自由建构」**：应试导向必须有课标锚定。学生漏的知识点自己建构不出来——「不知道自己不知道」是认知盲区，无法自愈。只有外部标准库才能告诉学生「你漏了这些」。

**为什么个人层不参与评分**：个人建构质量不可控，会污染五维计算；评分只看课标节点，才能保证跨学生可比。

### 2. 五维评分：课标锚定

K/A/T/Q/S 五维分数基于课标知识库的统一节点计算，跨学生可比。评分有三个支撑：

1. **课标库覆盖**：定义「应该掌握什么」的下限
2. **行为信号驱动**：对话、做题、迁移应用等行为产生五维分数
3. **做题数据校准**：真题/统考表现反向修正五维分数，防止「图谱自洽但应试失灵」

> 不采用「相对度量」（自我参照）。学生 A 在自己图谱里 K=100 但漏了课内知识点，实际比 B 的 K=80 差——相对分数会误导。高考是绝对比较，产品必须用绝对标准校准。

### 3. 课标库构建：三层叠加

不依赖单一来源，采用三层叠加构建：

| 层 | 来源 | 作用 | 工作量 |
|----|------|------|--------|
| 骨架 | 教材目录 | 提供章节结构 | 极低 |
| 考点 | 教辅书目录（如《五三》） | 细化到考点级 | 中（手工录入） |
| 校验 | 高考真题反推 + AI 校验 | 发现遗漏 + 标注权重 | 高（持续迭代） |

教材粒度是「章/节」，高考考的是「考点」，两者差 3-5 倍，光扒教材会漏大量细分考点。AI 列表约 30% 是噪音，可作为校验工具但不能作为构建主力。

### 4. 学生新增节点：AI 讨论伙伴

学生可在个人层新增节点（名称 + 父级），AI 作为**讨论伙伴**给出判断，学生可与 AI 适度讨论后确认添加：

1. 学生提交新节点（名称 + 父级）
2. AI 给出判断：是否为有效知识点？是否与课标节点重复？是否应归入其他父级？
3. 学生可与 AI 多轮讨论，AI 进一步引导
4. 学生最终确认添加 → 入库（无论 AI 是否赞同）

**AI 不拦截，只引导；最终决定权在学生。** 不设审核、不设兜底、不设防操纵机制——本系统的唯一用户门槛是「想学」，想学的学生不会恶意操纵评分。个人节点不参与五维评分（课标库是唯一评分锚点）。

### 5. 产品定位：助教 + 教练

本系统是**课内学习的助教 + 个人个性化教练**。

| 角色 | 职责 | 边界 |
|------|------|------|
| 助教 | 辅助课内学习，查漏补缺 | 不替代课堂，不替代老师 |
| 教练 | 个性化引导，帮学生看清自己 | 不替学生做决定，培养判断力 |

**用户门槛：想学。** 核心服务对象是「想学 + 判断力不足」的学生——工具为这部分人补位判断力，引导形成自我诊断能力。不想学的用户不是服务对象。本产品的本质是**弥补判断力差距**。

### 6. 悲观提醒哲学

局面过度乐观时，系统主动加入悲观提醒，防止给出虚假的「你很好」信号：

| 场景 | 过度乐观 | 悲观提醒 |
|------|---------|---------|
| K 分满分 | 「我全掌握了」 | 课标库可能不完整，满分不代表应试无懈可击 |
| 新增节点 | 「加了这个就完整了」 | 确认是新知识点，不是已有节点的别名？ |
| 连续学习 N 天 | 「我很努力」 | 做题正确率没提升，努力方向可能不对 |
| 五维全高分 | 「我很强」 | 检查是否题目太简单，或图谱有遗漏 |

**本质**：不打击学生，但防止系统比考试更温柔。应试的反馈是残酷的，产品不能给学生虚假的安全感。

### 7. 诚实声明（当前阶段）

- 当前为 **15 人小规模邀请使用**阶段，聚焦个体诊断，**数学单科突破**
- 五维评分以行为信号驱动的启发式估算为主，**做题模块尚未实现，课标锚定尚未完全建立**
- BKT/IRT 等标准化模型为远期目标

### 当前进度

数学课标知识库由项目负责人构建中，作为首个统一锚点。其他学科先搭建框架，后续逐步填充。

## 六、快速开始

### 环境要求

- 微信开发者工具（最新稳定版）
- 微信小程序 AppID
- 已开通微信云开发

### 运行步骤

1. **克隆仓库**

   ```bash
   git clone https://github.com/again-q/ai-learning-system.git
   ```

2. **导入项目**

   微信开发者工具 → 导入项目 → 选择仓库根目录 → 填入 AppID

3. **开通云开发**

   工具栏点击「云开发」按钮开通，记下**云环境 ID**

4. **配置云环境**

   编辑 [miniprogram/app.js](miniprogram/app.js)，替换 `env`：

   ```js
   wx.cloud.init({
     env: 'your-cloud-env-id',
     traceUser: true
   });
   ```

5. **部署云函数**

   右键 `cloudfunctions/userLogin` → 「上传并部署：云端安装依赖」

6. **初始化数据库**

   云开发控制台创建 `users` 集合（权限：仅创建者可读写）

7. **预览**

   点击工具栏「预览」或「真机调试」
