# 五维能力模型 — 学科掌握诊断框架

> 理论构建：again-q
> 版本：v3.0（2026-07-14）
> 状态：✅ 已定稿
> 定位：课后辅助诊断工具，不代替学校

---

## 一、核心命题

用一个可量化的、可操作的模型，让一个初高中生知道自己"到底哪里不会"、"为什么不会"、"下一步该做什么"。

三个子问题：
1. **"掌握"到底指什么？** — 不只是"会不会"，而是"会到什么程度"
2. **怎么把它变成可测的数字？** — 从学生日常做题数据中提取五个维度的信号
3. **测出来之后怎么用？** — 输出可操作的诊断报告和行动建议

---

## 二、核心 Pipeline

所有维度的计算基于统一的数据管道：

```
拍照 → AI 识别题目 + 学生作答 → 对比正确答案
    ↓
LLM 错因分类（将"错"拆为 4 类）
    ↓
计算失误/抄错数字           → S 维度扣分
套错公式但结构对             → A 维度扣分
题意理解偏/答非所问          → T/K 维度扣分
完全空白/乱填               → K 维度扣分
    ↓
    高价值题目？ → 弹窗问思维链 → Q 评分
    ↓
更新五维分数 → 输出诊断报告
```

**核心原则**：每个维度使用互不重叠的数据源，避免维度间的信号污染。

---

## 三、五维能力向量

```
M = (K, A, T, Q, S)
```

| 维度 | 对学生说 | 认知本质 | 粒度 | 数据源 | 覆盖率 |
|------|---------|---------|------|--------|--------|
| K | 知不知道？ | 陈述性记忆 | 知识点（树状聚合） | 难度×过程分（加权得分法） | 稀疏（仅回忆类题） |
| A | 会不会用？ | 程序性知识流畅度 | 教材单元 | D×η（有效刺激量），A/U框架 | 全量 |
| T | 能不能串？ | 跨单元结构识别 | 全局（每学生一个） | W×D×r，T/U框架 | 稀疏 |
| Q | 会不会学？ | 元认知（觉察与调整） | 全局 | 思维链（弹窗触发，独占） | 稀疏 |
| S | 稳不稳？ | 失常模式诊断 | 全局 | S0自动计算 + S1~S6回溯自报 | 全量 |

## 四、K — 知识掌握

### 4.1 本质定义

K（知识掌握） = 知识点在学生的长时记忆里是否存好。

它不是"会不会用"（那是A），也不是"能不能串"（那是T），而是单纯的"知不知道"——概念定义记住了吗？公式背熟了吗？性质理解了吗？

### 4.2 三层结构

| 层 | 内容 | 计入K总分 |
|---|------|---------|
| 知识层 | 概念定义、教材方法 | ✅ 是 |
| 模型层 | 题目模型、解题框架 | ❌ 参考输出，不参与评分 |
| 方法层 | 思维技巧、解题策略 | ❌ 参考输出，不参与评分 |

模型层和方法层作为参考信息输出，帮助诊断"学生知道概念但不会用模型"的情况，但不影响K值的计算。

### 4.3 树状结构

K维度的数据结构是一棵树：

```
学科
├── 单元
│   ├── 话题（如"二次函数"）
│   │   ├── 知识点A（如"顶点式"）
│   │   ├── 知识点B（如"一般式"）
│   │   └── 知识点C（如"交点式"）
│   └── 话题（如"一次函数"）
│       ├── 知识点D（如"斜率"）
│       └── 知识点E（如"截距"）
```

- **叶子节点**（精细知识点）：用加权得分法 S_k/D_k 计算掌握度
- **父节点**（话题/单元/学科）：用加权平均聚合子节点的掌握度

### 4.4 叶子节点算法：加权得分法

**基本公式**：
```
每道题的贡献 = 难度(D) × 过程分(P)
知识点累计得分 S_k = Σ 贡献
知识点累计难度 D_k = Σ 难度
掌握度 = S_k / D_k
```

**掌握度的本质**：掌握度 = S_k / D_k = Σ(D × P) / Σ D。这是一个加权平均——每道题的过程分 P，按难度 D 加权，求平均。换句话说：掌握度 = 所有做过题目的平均表现，但难题权重更高。

**难度 D 的判定（LLM，5维度加权平均）**：

| 维度 | 低分 | 高分 |
|------|------|------|
| 知识复杂度 | 单步套公式 | 多步+跨知识点 |
| 思维深度 | 直接套用 | 逆向/构造 |
| 陷阱密度 | 直白无坑 | 多陷阱/分类讨论 |
| 计算强度 | 口算级别 | 复杂运算 |
| 陌生度 | 常规题 | 新题型 |

D 的范围：0.01~0.99。

**过程分 P 的判定（LLM，4档）**：

| 分数 | 描述 |
|------|------|
| 1.0 | 完全正确，思路清晰 |
| 0.5 | 部分正确 / 正确但思路模糊 |
| 0.3 | 错误但有正确思路 |
| 0 | 完全错误 / 空白 |

**更新时机**：仅回忆类题（默写公式、复述定义、判断对错）触发K维度更新。应用类题不更新K。

**超纲规则**：LLM判定超纲的题目，做对了→正常更新（S_k和D_k都加）；做错了→不计入D_k（不降掌握度）。

### 4.5 父节点聚合：加权平均

**公式**：
```
父节点掌握度 = Σ(w_i × 子节点掌握度_i) / Σ w_i
```

**权重 w_i 的决定因素**：
- 知识点在话题中的核心程度（如"顶点式"比"一般式"在二次函数话题中更核心）
- 高考/大考的考查频次
- 学科大纲中的课时分配

权重由学科知识图谱内置，静态维护。

**聚合层级**：
```
学科总掌握度 = Σ(w_单元 × 单元掌握度) / Σ w_单元
单元掌握度 = Σ(w_话题 × 话题掌握度) / Σ w_话题
话题掌握度 = Σ(w_知识点 × 知识点掌握度) / Σ w_知识点
```

每一层的权重都由知识图谱内置，反映该节点在上一层中的重要程度。

### 4.6 输出格式

**4档制**：

| 档位 | 掌握度范围 | 说明 |
|------|-----------|------|
| 🟢 掌握 | ≥ 0.80 | 知识点牢固，可进入下一阶段 |
| 🟡 巩固 | 0.60~0.79 | 基本掌握，需要定期复习 |
| 🟠 薄弱 | 0.40~0.59 | 掌握不牢，需要针对性练习 |
| 🔴 紧急 | < 0.40 | 未掌握，需要重新学习 |

**话题级输出**：
- 话题整体掌握度 = 各知识点掌握度的加权平均
- 同步输出该话题下最薄弱的1~3个知识点

### 4.7 七要素归属

| 要素 | 归属维度 |
|------|---------|
| 定义、性质、操作、关系、表示 | K 维度 |
| 应用 | A 维度 |
| 来源 | Q 维度 |

### 4.8 参数总表

| 参数 | 值 | 说明 |
|------|-----|------|
| D范围 | 0.01~0.99 | 5维度加权平均 |
| P档位 | 1.0/0.5/0.3/0 | LLM判定 |
| 掌握档位边界 | ≥0.80掌握 / 0.60巩固 / 0.40薄弱 / <0.40紧急 | 4档制 |
| 父节点聚合 | 加权平均 | 权重由知识图谱内置 |

## 五、A — 应用能力

### 5.1 本质定义

A（应用能力） = 运用所学知识解题的能力指数。

它不是"做对的概率"，也不是"知识点的掌握度"，而是一个人运用所学知识解决问题的水平高低。

### 5.2 与K和T的区别

| 维度 | 本质 | 举例 |
|------|------|------|
| K | 知不知道这个公式（陈述性记忆） | 能默写F=ma |
| A | 能不能识别出该用这个公式并正确运用（程序性知识流畅度） | 看到斜面题知道用F=ma |
| T | 能不能把多个公式串起来（跨单元串联） | 把F=ma和匀变速直线运动公式结合起来解题 |

### 5.3 两层结构

| 粒度 | 名称 | 数量 | 含义 |
|------|------|------|------|
| 单元级 | 单元A | 每单元一个 | 该单元内的应用能力指数 |
| 全局级 | 全局A | 每学生一个 | 所有单元A加权聚合，反映整体应用能力水平 |

全局A不是独立于A的另一个维度，而是A维度在全局层面的聚合产物。

### 5.4 核心思想

1. A是能力指数，不是概率 — 它随着有效练习而增长，不会围绕一个固定值波动。
2. 学习是分单元进行的 — 每个单元独立维护自己的A和U，新单元的A从初始值开始。
3. A的增长受上限U约束 — A不能超过U，U本身是浮动的，由解题质量决定。
4. 路径质量决定增速和上限走向 — 暴力计算的人A涨得慢，U不涨甚至下浮；使用本质解法的人A涨得快，U持续上浮。

### 5.5 单元A核心算法

**每个单元维护的数据**：

| 变量 | 含义 | 初始值 | 范围 |
|------|------|--------|------|
| A | 当前能力指数 | 0.30 | [0, U] |
| U | 当前能力上限 | 0.50 | [A, 1] |
| N | 总观测次数 | 0 | 整数 |

**有效刺激量 E**：
```
E = D × η
```
其中：
- D（题目鉴别力）：LLM分析题目文本，复用K维度的5维度（知识复杂度、思维深度、陷阱密度、计算强度、陌生度），权重调整。范围0~1。
- η（路径质量系数）：LLM分析学生步骤文本，判断解法是否接近本质。范围0.4~1.0。

取值规则：
- 做对时：E > 0（正向刺激）
- 做错且是A问题：E < 0（负向刺激，取 -D × η）
- 做错且是K/S问题：E = 0（不更新A）

**上限U的更新**：
```
if E >= 0.5 and η >= 0.7:
    # 上浮：学生展示了本质解法，有突破潜力
    U += 0.05 × (1 - U)

elif 连续5道题 η <= 0.6:
    # 下浮：学生持续暴力计算，挤压虚假上限
    U -= 0.03 × (U - A)
```

上浮的语义：当学生展示出"这道题我用了接近本质的解法，且题本身鉴别力够"时，我们认为他有能力突破当前上限，U缓慢上浮。

下浮的语义：如果学生长期只用暴力计算，他的真实上限其实到不了U当前值——U应该向下收敛到A，把"虚假上限"挤掉。

**能力指数A的更新**：
```
if E > 0:
    ΔA = 0.25 × E × (U - A)
    A = min(A + ΔA, U)       # 硬约束：A不超过U

elif E < 0:
    ΔA = 0.25 × |E| × (U - A)
    A = max(A - ΔA, 0)       # 不低于0
```

ΔA = α × E × (U - A) 的含义：
- α（学习率）：控制更新灵敏度，α=0.25
- E（有效刺激量）：这次练习的质量
- (U - A)：剩余上升空间——A离上限还有多远
- 三个因子相乘：任何一个因子小，更新量就小；三个因子都大，更新量才大

**输出精度**：全精度计算，对外输出保留两位小数（百分位），第三位四舍五入。

### 5.6 单元A的行为特征

| 阶段 | N范围 | A的行为 | 原因 |
|------|-------|---------|------|
| 入门期 | 1~5 | 快速拉升，每跳0.02~0.04 | (U-A)大，E即使中等也能推动明显增长 |
| 快速拉升期 | 6~15 | 持续上涨，偶有震荡 | 高质量练习推动U上浮，A追赶 |
| 平台期 | 16~25 | 涨速放缓，A接近U | (U-A)变小，ΔA自然减小 |
| 突破期 | 26~35 | 突然跃升（开窍感） | 本质解法触发U上浮，A重新获得增长空间 |
| 精进期 | 36~50 | 微幅波动，缓慢爬升 | (U-A)趋近0，U涨速极慢 |
| 稳定期 | 50+ | 基本不动，收敛于U | A和U几乎重合，能力达到当前上限 |

### 5.7 全局A

**本质定义**：全局A = 由各单元A加权聚合而成的综合指标，反映学生整体的应用能力水平。

全局A较高不仅说明已学单元的应用能力强，也隐含了学新单元的迁移潜力（学习能力的间接体现）。

**各单元权重**：
```
w_i = importance_i × confidence_i × stage_i
```

- importance_i（静态）：由学科大纲内置，反映高考占比/单元核心度。不需要和为1，归一化时处理。
- confidence_i（动态）：由该单元的观测次数 N_i 决定。

```
confidence_i = 1 - e^{-0.1 × N_i}
N=10 → confidence≈0.63
N=20 → confidence≈0.86
N=50 → confidence≈0.99
```

- stage_i（动态）：由最近M次上传（M=20）中该单元题目的占比 active_i 决定。

```
stage_i = 0.7 + 1.0 × active_i
active_i = 0 → stage = 0.7（老单元基础权重）
active_i = 1.0 → stage = 1.7（正在密集学）
```

**不参与聚合的单元**：
- N_i = 0（完全没学过）→ confidence=0 → w_i=0
- 已结课且长期未碰的单元：active=0 → stage=0.7，仍有基础权重，不会完全消失

**聚合与平滑**：
```
w_i' = w_i / Σw_j          # 归一化
聚合值 = Σ(w_i' × A_i)     # 加权平均

全局A_new = 全局A_old + 0.25 × (聚合值 - 全局A_old)   # EWMA平滑
```

**初始值**：首个有数据的单元的A值。

**输出条件**：有效单元数 ≥ 2 且 总N ≥ 15 时，输出全局A。否则只输出当前单元A。

### 5.8 参数总表

| 参数 | 值 | 说明 |
|------|-----|------|
| α | 0.25 | 学习率 |
| δ_u | 0.05 | U上浮速率 |
| δ_d | 0.03 | U下浮速率 |
| λ | 0.1 | confidence(N)衰减率 |
| base | 0.7 | stage(active)老单元基础权重 |
| k | 1.0 | stage(active)活跃加成系数 |
| α_g | 0.25 | 全局A平滑率 |
| — | importance_i | 大纲内置，静态权重 |

## 六、T — 迁移能力

### 6.1 本质定义

T（迁移能力） = 学生识别跨单元深层结构、组合不同单元知识解题的全局能力。

它不是"在某个结构上的迁移能力"，而是这个人本身的属性——无论给他什么跨单元题，他都能更快地识别出深层结构。

### 6.2 核心属性

- **全局的**，不依赖单元（不像A每个单元一个）
- **初始值低**（0.10），因为跨单元综合题通常在学完多个单元后才出现
- **样本量少**（跨单元题远少于单元内题），所以更新更保守

### 6.3 与A维度的关系

| 维度 | 粒度 | 数量 | 初始值 | 更新触发 |
|------|------|------|--------|---------|
| A | 单元级 + 全局聚合 | 每单元一个A + 一个全局A | 0.30 | 单元内题 |
| T | 全局级 | 每学生一个 | 0.10 | 跨单元综合题 |

### 6.4 数据源

| 信号 | 来源 | 说明 |
|------|------|------|
| W（跨类型权重） | LLM根据知识图谱判断 | 反映跨单元的类型和难度 |
| D（题目鉴别力） | LLM分析题目文本 | 复用A维度的D |
| r（解题质量系数） | LLM多轮引导后评分 | 0~1，反映学生是否识别了深层结构 |
| T_shown | 从本次表现推断 | 由r和作答结果共同决定 |

### 6.5 核心算法

**每个学生维护的数据**：

| 变量 | 含义 | 初始值 | 范围 |
|------|------|--------|------|
| T | 当前迁移能力指数 | 0.10 | [0, TU] |
| TU | 当前迁移能力上限 | 0.60 | [T, 1] |

**有效刺激量 E_t**：
```
E_t = W × D × r
```

**T_shown（展现出的能力）**：T_shown由LLM根据学生的作答结果和思维链综合判断。它不是"当前T值"，而是"这次表现展现出的能力水平"。

| 表现 | T_shown |
|------|---------|
| 竞赛压轴题做对 + 说清结构 | 0.95~0.98 |
| 高考压轴题做对 + 说清结构 | 0.80 |
| 高考压轴题做对 + 说不清结构 | 0.50 |
| 部分对 + 结构方向对 | 0.30 |
| 完全做错 + 结构没认出 | 0.05 |

**TU的更新**：

上浮触发（两个条件同时满足）：
1. 最近3道跨单元题中，有任意一道的 T_shown > TU × 1.1
2. 最近3道跨单元题的 T_shown 平均值 > TU

上浮幅度：
```
TU_new = TU_old + 0.1 × (T_shown_max - TU_old)
```

下浮触发：最近3道跨单元题的 T_shown 平均值 < TU × 0.7

下浮幅度：
```
TU_new = TU_old - 0.09 × (TU_old - avg_T_shown)
```

**T的更新（预期计算）**：T的更新采用预期偏差机制——将T_shown（这次展现的能力）与当前T比较，决定涨跌方向。

先更新TU，再用新TU计算ΔT。

```
ΔT = 0.20 × (T_shown - T) × (TU_new - T)
T_new = clamp(T + ΔT, 0, TU_new)
```

- 如果 T_shown > T → T上涨（这次表现超出当前水平）
- 如果 T_shown < T → T下跌（这次表现低于当前水平）
- 如果 T_shown ≈ T → T不变或微调

为什么用预期偏差？T的样本量少，每道跨单元题都很珍贵。如果只用"做对就涨、做错就跌"的二元逻辑，会浪费很多信息——比如一道难题做错了但结构方向对，应该比"完全没认出结构"跌得少。预期偏差机制可以更精细地利用每次观测。

### 6.6 W（跨类型权重）分段

| 类型 | W范围 | 说明 |
|------|-------|------|
| 课内跨单元（多单元、难单元） | 0.60~0.80 | 核心考察对象 |
| 跨学科 | 0.50~0.60 | 按难度细分为0.50/0.55/0.60 |
| 初中跨高中 | 0.30~0.50 | 工具性迁移，参考度低 |
| 竞赛 | 0.80~0.999999 | 顶级迁移，无限逼近1但不到1 |

### 6.7 r（解题质量系数）的判定

r由LLM通过多轮引导后评分。LLM根据学生的回答，判断他是否识别了跨单元的深层结构。

| 分数 | 描述 |
|------|------|
| 0.9~1.0 | 学生能在追问下清晰说出"为什么这两个单元能连在一起"，并能指出是哪个条件触发了他这个联想 |
| 0.7~0.8 | 学生能说出涉及哪些单元，但说不清为什么能连在一起 |
| 0.4~0.6 | 学生一开始没看出来，但在追问下能事后领悟 |
| 0.1~0.3 | 学生全程说不清，只停留在套公式层面 |
| 0 | 学生拒绝回答，或思维链完全空白 |

### 6.8 T维度的行为特征

| 阶段 | T的行为 | 原因 |
|------|---------|------|
| 初期（N<5） | 快速拉升，每跳0.05~0.10 | TU初始值0.60，T初始值0.10，(TU-T)大 |
| 中期（N=5~15） | 震荡上行，涨速放缓 | T接近TU，(TU-T)变小 |
| 突破期 | 突然跃升 | 高质量跨单元题推动TU上浮，T获得新增长空间 |
| 后期（N>15） | 稳定在高位，微幅波动 | T和TU几乎重合 |
| 低谷期 | T下跌 | 持续表现不佳，T_shown低于当前T |

### 6.9 参数总表

| 参数 | 值 | 说明 |
|------|-----|------|
| T₀ | 0.10 | 初始迁移能力 |
| TU₀ | 0.60 | 初始迁移上限 |
| α_t | 0.20 | 学习率 |
| δ_up | 0.10 | TU上浮速率 |
| δ_down | 0.09 | TU下浮速率 |
| N | 3 | TU上下浮观测窗口 |
| 上浮条件 | T_shown_max > TU×1.1 且 avg_T_shown > TU | 突破+趋势确认 |
| 下浮条件 | avg_T_shown < TU×0.7 | 持续低迷 |

## 七、Q — 思维品质

### 7.1 本质定义

Q（思维品质） = 觉察和调整的能力——学生知不知道自己是怎幺错的，下次能不能改过来。

通俗说法："会不会学"。

理论依据：Flavell 元认知理论（监测、控制、评估）。

### 7.2 核心原则

1. Q只通过思维链测量。没有思维链，Q不更新。
2. 思维链不让学生主动写，而是AI判断为高价值题目后弹窗询问。
3. 对题也能涨Q（思路清晰说明觉察能力强）。

### 7.3 五个子维度

| 编号 | 子维度 | 含义 | 对应 Flavell 成分 |
|------|--------|------|------------------|
| Q1 | 事中监控 | 做题过程中能否及时察觉"卡住了/偏了" | 监测（事中） |
| Q2 | 归因准确度 | 事后能否准确说出错因/思路偏差 | 监测（事后） |
| Q3 | 调整清晰度 | 能否说出具体的改进策略 | 控制 |
| Q4 | 反思深度 | 能否提炼通用经验、归类错误模式 | 评估（过程） |
| Q5 | 执行结果 | 下次同类题是否做对了 | 评估（结果） |

### 7.4 数据采集

**触发方式**：AI识别高价值题目 → 弹窗问"你是怎么想的？"

**高价值题目定义**：
- 做错的题
- 做对但过程复杂的题
- 涉及关键知识节点的题

**弹窗频率自适应**：
```
弹窗概率 = base_prob × (1 - Q)
```
- Q低（0.30）→ 弹窗概率高（几乎每题高价值题都弹）
- Q高（0.80）→ 弹窗概率低（鼓励主动写思维链，系统少打扰）
- base_prob = 0.7~0.8。

### 7.5 评分方式

LLM根据学生回答，给每个子维度打0~1分。

| 子维度 | 评分标准 |
|--------|---------|
| Q1 事中监控 | 0=无察觉，0.5=事后回想起来有，1=过程中主动说"这里不对" |
| Q2 归因准确度 | 0=说不清，0.5=方向对但不准，1=精准 |
| Q3 调整清晰度 | 0=无，0.5=泛泛（"下次细心"），1=具体可执行 |
| Q4 反思深度 | 0=无，0.5=单次改进，1=模式归纳 |
| Q5 执行结果 | 首次思维链时占位0.5，下次同类题验证后更新为1.0（做对）或0（做错） |

### 7.6 Q值的计算

**数据结构**：

| 字段 | 含义 | 初始值 | 范围 |
|------|------|--------|------|
| Q | 全局思维品质指数 | 0.30 | [0, 1] |
| Q1~Q5 | 5个子维度的移动平均 | 0.30 | [0, 1] |
| N | 有效思维链总次数 | 0 | 整数 |

**子维度的更新（移动平均）**：
```
Q_i_new = (Q_i_old × N + s_i) / (N + 1)   （N ≥ 3）
```

冷启动（N < 3）时放大步长，让Q快速收敛到真实水平：
```
Q_i_new = (Q_i_old × 1 + s_i) / 2
```

N在所有子维度上共用（每次有效思维链，N+1）。

**全局Q的聚合**：
```
Q = 0.25×Q1 + 0.25×Q2 + 0.25×Q3 + 0.15×Q4 + 0.10×Q5
```

权重分配的依据：

| 子维度 | 权重 | 理由 |
|--------|------|------|
| Q1 事中监控 | 0.25 | 监测是元认知的起点，事中比事后更稀缺 |
| Q2 归因准确度 | 0.25 | 和事中监控对等，合起来0.5覆盖监测 |
| Q3 调整清晰度 | 0.25 | 控制是把监测转化为行动的关键 |
| Q4 反思深度 | 0.15 | 反思偏高阶，出现频率低，权重稍小 |
| Q5 执行结果 | 0.10 | 结果验证，是Q1~Q4的外部校准，权重最小 |

**Q5的特殊处理（二次更新）**：Q5和其他四个子维度不同——它不能在第一次思维链时就给分，因为"下次同类题做对没"是未来事件。

处理流程：
1. 首次思维链（做错当前题）：Q1~Q4正常给分，Q5先给 0.5（占位）
2. 下次遇到同类题（由LLM根据知识图谱判断）：
   - 做对了 → Q5更新为1.0
   - 做错了 → Q5更新为0
   - 更新方式：Q5_new = (Q5_old × N + 新Q5) / (N + 1)

### 7.7 算例

一个学生第5次有效思维链，各子维度当前值：Q1=0.70, Q2=0.65, Q3=0.55, Q4=0.45, Q5=0.60，N=4。

本次思维链评分：s1=0.8, s2=0.7, s3=0.6, s4=0.5, s5=0.5（占位）。

N=4 ≥ 3，用标准移动平均：

```
Q1_new = (0.70×4 + 0.8) / 5 = 3.6/5 = 0.72
Q2_new = (0.65×4 + 0.7) / 5 = 3.3/5 = 0.66
Q3_new = (0.55×4 + 0.6) / 5 = 2.8/5 = 0.56
Q4_new = (0.45×4 + 0.5) / 5 = 2.3/5 = 0.46
Q5_new = (0.60×4 + 0.5) / 5 = 2.9/5 = 0.58
N = 5

全局Q = 0.25×0.72 + 0.25×0.66 + 0.25×0.56 + 0.15×0.46 + 0.10×0.58
     = 0.180 + 0.165 + 0.140 + 0.069 + 0.058 = 0.612
```

### 7.8 参数总表

| 参数 | 值 | 说明 |
|------|-----|------|
| 初始Q | 0.30 | 默认值 |
| base_prob | 0.7~0.8 | 弹窗基础概率 |
| 冷启动阈值 | N < 3 | 放大更新步长 |
| Q1权重 | 0.25 | 事中监控 |
| Q2权重 | 0.25 | 归因准确度 |
| Q3权重 | 0.25 | 调整清晰度 |
| Q4权重 | 0.15 | 反思深度 |
| Q5权重 | 0.10 | 执行结果 |
| Q5首次占位 | 0.5 | 等待下次同类题验证 |

## 八、S — 执行稳定性

### 8.1 本质定义

S（执行稳定性） = 学生实际表现与真实能力之间的偏差。

它不是"会不会"（那是K/A/T），而是"稳不稳"——该做对的题做错了，不该错的错了，明明会做的因为计算、审题、状态、心态等原因丢了分。

### 8.2 核心属性

- **全局的**，一个学生一个S
- **不是能力指数**，不搞"增长+上限"那套
- **是失常模式的统计归档**，服务于诊断和干预

### 8.3 七个子维度

| 编号 | 子维度 | 含义 | 数据来源 | 性质 | 是否计入S_global |
|------|--------|------|---------|------|----------------|
| S0 | 波动指数 | 预期得分率与实际得分率的绝对偏差 | K/A/T综合计算 | 客观 | ✅ 是 |
| S1 | 计算稳定性 | 计算过程是否可靠 | 思维链回溯 | 主观 | ✅ 是 |
| S2 | 时间管理 | 时间压力下是否稳定 | 思维链回溯 | 主观 | ✅ 是 |
| S3 | 审题稳定性 | 审题是否仔细 | 思维链回溯 | 主观 | ✅ 是 |
| S4 | 步骤稳定性 | 步骤执行是否流畅 | 思维链回溯 | 主观 | ✅ 是 |
| S5 | 状态稳定性 | 做题前的生理/心理状态 | 回溯结束时轻量自评 | 主观（归因） | ❌ 否 |
| S6 | 考场心态 | 考试过程中的心理波动 | 考试级上传时询问 | 主观（归因） | ❌ 否 |

### 8.4 S0波动指数算法

**单次偏差 d**：
```
P_exp = w_K × K_mastery + w_A × A_unit + w_T × T_global
```

权重由题目类型决定：
- 单元内题：w_K=0.3, w_A=0.7, w_T=0
- 跨单元题：w_K=0.2, w_A=0.4, w_T=0.4

```
R_actual = 学生实际得分 / 该题满分
d = |P_exp - R_actual|
```

d的范围：0~1。d越大，说明实际表现与预期偏差越大。

**波动指数 V**：
```
V = 最近N次d的平均值
```
N = 20~30次作答。

**各维度波动**：
```
V_K = 最近N次d_K的平均值（仅当该题涉及K偏差时）
V_A = 最近N次d_A的平均值（仅当该题涉及A偏差时）
V_T = 最近N次d_T的平均值（仅当该题涉及T偏差时）
```
其中d_K, d_A, d_T由LLM从错因分析中拆解。

**关于超常发挥的处理**：S0使用绝对偏差 d = |P_exp - R_actual|，因此无论是失常（预期高但实际低）还是超常（预期低但实际高），只要有偏差，都计入波动指数。这是因为"超常"本身也是一种不稳定——一个学生如果经常"该做对的做错，不该做对的蒙对"，他的发挥就是不可预测的，这本身就是S要反映的问题。

### 8.5 全局稳定性指数 S_global

**公式**：
```
S_global = w_self × Self_stability + w_obj × Obj_stability
```
其中：
- Self_stability = 1 - Σ(w_i × 失常率_i) （自报部分，来自S1~S4）
- Obj_stability = 1 - V （客观部分，来自S0波动指数）
- w_self = 0.6
- w_obj = 0.4

**Self_stability 的详细计算**：
```
失常率_i = S_i的失常次数 / S_i的总观测次数（i=1~4）
w_i（动态权重）= 失常率_i / Σ(失常率_j)
```
失常率越高的子维度，权重越大。这样S_global会重点反映学生最薄弱的稳定性环节。

**Obj_stability 的详细计算**：
```
Obj_stability = 1 - V
```

**算例**：

一个学生：
- S1失常率=0.35，S2失常率=0.05，S3失常率=0.12，S4失常率=0.08
- V = 0.18

Step 1：Self_stability

Σ失常率 = 0.35+0.05+0.12+0.08 = 0.60

w_i：
- S1: 0.35/0.60 = 0.583
- S2: 0.05/0.60 = 0.083
- S3: 0.12/0.60 = 0.200
- S4: 0.08/0.60 = 0.133

Σ(w_i × 失常率_i) = 0.583×0.35 + 0.083×0.05 + 0.200×0.12 + 0.133×0.08
= 0.204 + 0.004 + 0.024 + 0.011 = 0.243

Self_stability = 1 - 0.243 = 0.757

Step 2：Obj_stability

Obj_stability = 1 - 0.18 = 0.820

Step 3：S_global

S_global = 0.6 × 0.757 + 0.4 × 0.820 = 0.454 + 0.328 = 0.782

### 8.6 S与K/A/T的边界

| 情况 | 归属 |
|------|------|
| K/A/T达标，做错了 | S问题（失常） |
| K/A/T达标，做对了但步骤异常 | S问题（隐性损耗） |
| K/A/T不达标，做错了 | K/A/T问题，不进S |
| K/A/T不达标，做对了（侥幸） | K/A/T问题，不进S |
| 预期低但实际高（超常） | S0计入波动（d>0），但不视为失常 |

### 8.7 输出格式

```
稳定性指数：0.78（⭐⭐⭐⭐）

> 波动指数：0.18（较稳）

> 失常画像：
- 计算类失常率 35%（主要问题）→ 建议反向验算
- 审题类失常率 12% → 建议圈画条件
- 时间类失常率 5%
- 步骤类失常率 8%

> 状态记录：
- 最近7天，有3天报了"不太好"，这3天的失常率明显偏高
  （平均45% vs 平时的20%）。建议关注睡眠质量和情绪管理。

> 心态记录：
- 最近3场考试中，有1场心态崩盘，该场失常率高达60%。
  建议进行模拟考脱敏训练。
```

### 8.8 指导策略

| 子维度 | 失常率高时的建议 |
|--------|----------------|
| S0 波动指数高 | 分析偏差来源——是集中在某个单元、某种题型，还是时间压力下？ |
| S1 计算稳定性 | 每题做完后反向验算；草稿纸分区书写 |
| S2 时间管理 | 限时训练；学会"跳做→回头"的节奏 |
| S3 审题稳定性 | 读题时圈画关键条件；慢读一遍再动笔 |
| S4 步骤稳定性 | 关键步骤停顿确认；写完一步回头看一步 |
| S5 状态稳定性 | 保持规律作息；考前调整生物钟 |
| S6 考场心态 | 模拟考脱敏训练；深呼吸/正念练习 |

## 九、各维度数据源独立性（非重叠原则）

### 9.1 核心原则

**每个维度有且只有自己独占的信号源，一个信号不能同时用于计算两个维度。**

### 9.2 信号分配

| 维度 | 独占信号 | 不使用的信号 |
|------|---------|------------|
| K | 难度×过程分 | 无（加权得分法无外部参数） |
| A | D×η（有效刺激量），A/U框架 | 不直接使用对/错结果 |
| T | W×D×r（跨类型×鉴别力×解题质量），T/U框架 | 不处理单点对错 |
| Q | 思维链文本（弹窗触发，独占5维评分） | 不接受任何代理指标 |
| S | S0波动指数（客观）+ S1~S4失常自报（主观） | 不判断"这是粗心还是不会" |

### 9.3 跨维度参数传递

K维度的加权得分法不需要外部参数。

```
K的计算只依赖于：
  难度 0.01-0.99 ← LLM判定（5维度加权）
  过程分 0-1    ← LLM判定（4档）
  不需要 P(S)、P(G)、P(L₀)、P(T) 等参数
```

K向其他维度传递的信号：
- 具体到题目的过程分和思维链 → 提供给Q维度
- 超纲/不超纲标记 → 提供给学习进度指标

**五维不重叠**：每个维度有自己独占的信号源，不存在两个维度用同一数据算不同分的情况。
