## 一、核心命题

用一个可量化的、可操作的模型，让一个初高中生知道自己"到底哪里不会"、"为什么不会"、"下一步该做什么"。

三个子问题：
1. **"掌握"到底指什么？**—— 不只是"会不会"，而是"会到什么程度"
2. **怎么把它变成可测的数字？** —— 从学生日常做题数据中提取五个维度的信号
3. **测出来之后怎么用？** —— 输出可操作的诊断报告和行动建议

---

## 二、核心 Pipeline

所有维度的计算基于统一的数据管道：

```
拍照 → AI 识别题目 + 学生作答 → 对比正确答案
    ↓
LLM 错因分类（将"错"拆为 4 类）
    ↓
计算失误/抄错数字 → S 维度扣分
套错公式但结构对   → A 维度扣分
题意理解偏/答非所问 → T/K 维度扣分
完全空白/乱填      → K 维度扣分
    ↓
    高价值题目？ → 弹窗问思维链 → Q 评分
    ↓
更新五维分数 → 输出诊断报告
```

**核心原则**：每个维度使用互不重叠的数据源，避免维度间的信号污染。

---

## 三、五维能力向量

```
M = (K, A, T, Q, S)
```

| 维度 | 对学生说 | 认知本质 | 粒度 | 数据源 | 覆盖率 |
|------|---------|---------|------|--------|--------|
| K | 知不知道？ | 陈述性记忆 | 知识点 | 难度×过程分（加权得分法） | 稀疏（仅回忆类题） |
| A | 会不会用？ | 程序性知识流畅度 | 教材单元 | D×η（有效刺激量），A/U框架 | 全量 |
| T | 能不能串？ | 跨单元结构识别 | 全局（每学生一个） | W×D×r，T/U框架 | 稀疏 |
| Q | 会不会学？ | 元认知（觉察与调整） | 全局 | 思维链（弹窗触发，独占） | 稀疏 |
| S | 稳不稳？ | 失常模式诊断 | 全局 | S0自动计算 + S1~S6回溯自报 | 全量 |

### K — 知识掌握

**本质**：知识点在你的长时记忆里存好了吗？

**通俗说法**："知不知道"

**三层结构**：
| 层 | 内容 | 计入K总分 |
|---|------|---------|
| 知识层 | 概念定义、教材方法 | ✅ 是 |
| 模型层 | 题目模型、解题框架 | ❌ 参考 |
| 方法层 | 思维技巧、解题策略 | ❌ 参考 |

**树状结构**：知识点 → 话题 → 单元 → 学科
- **叶子节点**（精细知识点）：用加权得分法 S_k/D_k 计算
- **父节点**（话题/单元/学科）：用加权平均聚合子节点掌握度，权重由知识图谱内置（高考占比、核心程度）

**算法**：加权得分法（S_k / D_k）

**基本公式**：
```
每道题贡献 = 难度(0.01-0.99) × 过程分(0/0.3/0.5/1.0)
知识点累计得分 S_k = Σ 贡献
知识点累计难度 D_k = Σ 难度
掌握度 = S_k / D_k
```

**难度判定（LLM，5维度加权平均）**：
| 维度 | 低分 | 高分 |
|------|------|------|
| 知识复杂度 | 单步套公式 | 多步+跨知识点 |
| 思维深度 | 直接套用 | 逆向/构造 |
| 陷阱密度 | 直白无坑 | 多陷阱/分类讨论 |
| 计算强度 | 口算级别 | 复杂运算 |
| 陌生度 | 常规题 | 新题型 |

**过程分判定（LLM）**：
- 1.0 → 完全正确，思路清晰
- 0.5 → 部分正确 / 正确但思路模糊
- 0.3 → 错误但有正确思路
- 0 → 完全错误 / 空白

**超纲规则**：LLM 判定超纲的题目，做错时不降掌握度（不计入 D_k）；做对了正常更新。

**更新时机**：仅回忆类题（默写公式、复述定义、判断对错）触发 K 维度更新。应用类题不更新 K。

**遗忘机制**：超过 14 天未练习触发。
- 有 Q 维度理解标记的：掌握度 × 0.95
- 无 Q 维度理解标记的：掌握度 × 0.90

**话题级聚合**：
- 话题（如"二次函数"）下各知识点独立评分
- 话题整体掌握度 = 各知识点掌握度的加权平均
- 同步展示该话题下最薄弱的 1-3 个知识点

**展示**：4 档（掌握/巩固/薄弱/紧急） + 学习进度指标（待实现）

| 档位 | 掌握度范围 | 说明 |
|------|-----------|------|
| 🟢 掌握 | ≥ 0.80 | 知识点牢固，可进入下一阶段 |
| 🟡 巩固 | 0.60~0.79 | 基本掌握，需要定期复习 |
| 🟠 薄弱 | 0.40~0.59 | 掌握不牢，需要针对性练习 |
| 🔴 紧急 | < 0.40 | 未掌握，需要重新学习 |

**K 维度完整决策记录**：
| 决策点 | 结论 |
|--------|------|
| 底层算法 | 加权得分法（非 BKT），S_k/D_k |
| 过程分 | LLM 4 档（1.0/0.5/0.3/0） |
| 难度 | LLM 5 维度判定，0.01-0.99 连续值 |
| 超纲 | 超纲做错不计入 D_k |
| 遗忘 | 14 天触发，有 Q 理解标记衰减减半 |
| P(T) 学习率 | 不使用（只诊断不教学） |
| 话题级聚合 | 加权平均 + 展示最薄弱 |
| 模型层/方法层 | 参考展示，不参与评分 |
| 4 档边界 | ≥0.80掌握 / 0.60巩固 / 0.40薄弱 / <0.40紧急 |
| 更新时机 | 仅回忆类题触发，应用类题不更新K |

**七要素归属**：
| 要素 | 归属维度 |
|------|---------|
| 定义、性质、操作、关系、表示 | K 维度 |
| 应用 | A 维度 |
| 来源 | Q 维度 |

### A — 应用能力

**本质**：运用所学知识解题的能力指数。不是"做对的概率"，而是运用知识解决问题的水平高低。

**通俗说法**："会不会用"

**粒度**：教材单元（如"牛顿第二定律"一个单元维护一个 A）

**核心区别**：
```
K = 知不知道这个公式（陈述性记忆）
A = 能不能识别出该用这个公式并正确运用（程序性知识流畅度）
T = 能不能把多个公式串起来（跨单元串联）
```

**核心思想**：
1. A 是能力指数，不是概率 — 随着有效练习增长，不会围绕固定值波动
2. 学习是分单元进行的 — 每个单元独立维护自己的 A 和 U
3. A 的增长受上限 U 约束 — A 不能超过 U，U 本身由解题质量决定
4. 路径质量 η 决定增速和上限走向

**两层结构**：

| 粒度 | 名称 | 数量 | 含义 |
|------|------|------|------|
| 单元级 | 单元A | 每单元一个 | 该单元内的应用能力指数 |
| 全局级 | 全局A | 每学生一个 | 所有单元A加权聚合，反映整体应用能力 |

### 单元A核心算法

**每个单元维护的数据**：

| 变量 | 含义 | 初始值 | 范围 |
|------|------|--------|------|
| A | 当前能力指数 | 0.30 | [0, U] |
| U | 当前能力上限 | 0.50 | [A, 1] |
| N | 总观测次数 | 0 | 整数 |

**有效刺激量 E**：
```
E = D × η
D = 题目鉴别力（LLM复用K维度5维度，权重调整，0~1）
η = 路径质量系数（LLM分析步骤文本，判断解法是否接近本质，0.4~1.0）

做对时：E > 0（正向刺激）
做错且是A问题：E < 0（负向刺激，取 -D×η）
做错且是K/S问题：E = 0（不更新A）
```

**上限U的更新**：
```
if E >= 0.5 and η >= 0.7:
    # 上浮：学生展示了本质解法，有突破潜力
    U += 0.05 × (1 - U)
elif 连续5道题 η <= 0.6:
    # 下浮：学生持续暴力计算，挤压虚假上限
    U -= 0.03 × (U - A)
```

**能力指数A的更新**：
```
if E > 0:
    ΔA = 0.25 × E × (U - A)
    A = min(A + ΔA, U)       # 硬约束：A不超过U
elif E < 0:
    ΔA = 0.25 × |E| × (U - A)
    A = max(A - ΔA, 0)       # 不低于0
```

**展示**：两位小数（百分位）

### 全局A

全局A = 各单元A加权聚合，经EWMA平滑。

**权重公式**：
```
w_i = importance_i × confidence_i × stage_i
importance_i = 大纲内置（静态，反映高考占比）
confidence_i = 1 - e^{-0.1 × N_i}（动态，数据量越大越可信）
stage_i = 0.7 + 1.0 × active_i（动态，正在学的单元权重更高）
```

**平滑**：EWMA，α_g = 0.25

**展示条件**：有效单元数 ≥ 2 且 总N ≥ 15

**数据来源**：D（题目鉴别力，LLM复用K维度）、η（路径质量，LLM分析步骤文本）、对/错+错因（区分A/K/S问题）

### T — 迁移能力

**本质**：学生识别跨单元深层结构、组合不同单元知识解题的全局能力。不是"在某个结构上的迁移能力"，而是这个人本身的属性。

**通俗说法**："能不能串"

**核心属性**：
- **全局的**，不依赖单元（不像A每个单元一个）
- **初始值低**（0.10），因为跨单元综合题通常在学完多个单元后才出现
- **样本量少**（跨单元题远少于单元内题），所以更新更保守

**与A维度的关系**：

| 维度 | 粒度 | 数量 | 初始值 | 更新触发 |
|------|------|------|--------|---------|
| A | 单元级 + 全局聚合 | 每单元一个A + 一个全局A | 0.30 | 单元内题 |
| T | 全局级 | 每学生一个 | 0.10 | 跨单元综合题 |

#### 数据源

| 信号 | 来源 | 说明 |
|------|------|------|
| W（跨类型权重） | LLM根据知识图谱判断 | 跨单元0.6~0.8，竞赛0.8~0.999999 |
| D（题目鉴别力） | LLM分析题目文本 | 复用A维度的D |
| r（解题质量系数） | LLM多轮引导后评分 | 0~1，反映学生是否识别了深层结构 |
| T_shown | 从本次表现推断 | 由r和作答结果共同决定 |

#### 核心算法

**每个学生维护的数据**：

| 变量 | 含义 | 初始值 | 范围 |
|------|------|--------|------|
| T | 当前迁移能力指数 | 0.10 | [0, TU] |
| TU | 当前迁移能力上限 | 0.60 | [T, 1] |

**有效刺激量 E_t**：
```
E_t = W × D × r
```

**T_shown（展现出的能力）**：LLM根据作答结果和思维链综合判断。

| 表现 | T_shown |
|------|---------|
| 竞赛压轴题做对 + 说清结构 | 0.95~0.98 |
| 高考压轴题做对 + 说清结构 | 0.80 |
| 高考压轴题做对 + 说不清结构 | 0.50 |
| 部分对 + 结构方向对 | 0.30 |
| 完全做错 + 结构没认出 | 0.05 |

**TU的更新**：

上浮（两个条件同时满足）：
```
1. 最近3道跨单元题中，有任意一道的 T_shown > TU × 1.1
2. 最近3道跨单元题的 T_shown 平均值 > TU
TU_new = TU_old + 0.1 × (T_shown_max - TU_old)
```

下浮（条件）：
```
最近3道跨单元题的 T_shown 平均值 < TU × 0.7
TU_new = TU_old - 0.09 × (TU_old - avg_T_shown)
```

**T的更新**：
```
ΔT = 0.20 × (T_shown - T) × (TU_new - T)
T_new = clamp(T + ΔT, 0, TU_new)
```

#### 参数总表

| 参数 | 值 | 说明 |
|------|-----|------|
| T₀ | 0.10 | 初始迁移能力 |
| TU₀ | 0.60 | 初始迁移上限 |
| α_t | 0.20 | 学习率 |
| δ_up | 0.10 | TU上浮速率 |
| δ_down | 0.09 | TU下浮速率 |
| N | 3 | TU上下浮观测窗口 |

**r（解题质量系数）判定**：LLM多轮引导后评分

| 分数 | 描述 |
|------|------|
| 0.9~1.0 | 能在追问下清晰说出"为什么这两个单元能连在一起"，指出触发条件 |
| 0.7~0.8 | 能说出涉及哪些单元，但说不清为什么能连 |
| 0.4~0.6 | 一开始没看出来，在追问下能事后领悟 |
| 0.1~0.3 | 全程说不清，只停留在套公式层面 |
| 0 | 拒绝回答或思维链完全空白 |

**W（跨类型权重）分段**：

| 类型 | W范围 | 说明 |
|------|-------|------|
| 课内跨单元（多单元、难单元） | 0.60~0.80 | 核心考察对象 |
| 跨学科 | 0.50~0.60 | 按难度细分 |
| 初中跨高中 | 0.30~0.50 | 工具性迁移，参考度低 |
| 竞赛 | 0.80~0.999999 | 顶级迁移，无限逼近1但不到1 |

### Q — 思维品质

**本质**：觉察和调整的能力 — 学生知不知道自己是怎幺错的，下次能不能改过来。

**通俗说法**："会不会学"

**理论依据**：Flavell 元认知理论（监测、控制、评估）。

**核心原则**：
- Q 只通过思维链测量。没有思维链，Q 不更新。
- 思维链不让学生主动写，而是 AI 判断为高价值题目后弹窗询问。
- 对题也能涨 Q（思路清晰说明觉察能力强）。

#### 五个子维度

| 编号 | 子维度 | 含义 | 对应 Flavell 成分 |
|------|--------|------|------------------|
| Q1 | 事中监控 | 做题过程中能否及时察觉"卡住了/偏了" | 监测（事中） |
| Q2 | 归因准确度 | 事后能否准确说出错因/思路偏差 | 监测（事后） |
| Q3 | 调整清晰度 | 能否说出具体的改进策略 | 控制 |
| Q4 | 反思深度 | 能否提炼通用经验、归类错误模式 | 评估（过程） |
| Q5 | 执行结果 | 下次同类题是否做对了 | 评估（结果） |

#### 数据采集

**触发方式**：AI 识别高价值题目 → 弹窗问"你是怎么想的？"

**高价值题目**：做错的题、做对但过程复杂的题、涉及关键知识节点的题。

**弹窗频率自适应**：
```
弹窗概率 = base_prob × (1 - Q)
base_prob = 0.7~0.8

Q 低（0.30）→ 几乎每题都弹
Q 高（0.80）→ 少打扰，鼓励主动写
```

#### 评分方式

LLM 根据学生回答，给每个子维度打 0~1 分：

| 子维度 | 评分标准 |
|--------|---------|
| Q1 事中监控 | 0=无察觉，0.5=事后回想起来有，1=过程中主动说"这里不对" |
| Q2 归因准确度 | 0=说不清，0.5=方向对但不准，1=精准 |
| Q3 调整清晰度 | 0=无，0.5=泛泛（"下次细心"），1=具体可执行 |
| Q4 反思深度 | 0=无，0.5=单次改进，1=模式归纳 |
| Q5 执行结果 | 首次思维链占位0.5，下次同类题验证后更新 |

#### Q 值计算

**子维度更新（移动平均）**：
```
Q_i_new = (Q_i_old × N + s_i) / (N + 1)   （N ≥ 3）
冷启动（N < 3）：Q_i_new = (Q_i_old × 1 + s_i) / 2
```
N 在所有子维度共用（每次有效思维链，N+1）。

**全局 Q 聚合**：
```
Q = 0.25×Q1 + 0.25×Q2 + 0.25×Q3 + 0.15×Q4 + 0.10×Q5
```

**Q5 二次更新**：
1. 首次思维链（做错当前题）：Q1~Q4 正常给分，Q5 先给 0.5（占位）
2. 下次遇到同类题（LLM 根据知识图谱判断）：
   - 做对了 → Q5 更新为 1.0
   - 做错了 → Q5 更新为 0
   - 更新方式：Q5_new = (Q5_old × N + 新Q5) / (N + 1)

#### 数据结构

| 字段 | 含义 | 初始值 | 范围 |
|------|------|--------|------|
| Q | 全局思维品质指数 | 0.30 | [0, 1] |
| Q1~Q5 | 5个子维度的移动平均 | 0.30 | [0, 1] |
| N | 有效思维链总次数 | 0 | 整数 |

#### 参数总表

| 参数 | 值 | 说明 |
|------|-----|------|
| 初始 Q | 0.30 | 默认值 |
| base_prob | 0.7~0.8 | 弹窗基础概率 |
| 冷启动阈值 | N < 3 | 放大更新步长 |
| Q1/Q2/Q3/Q4/Q5 权重 | 0.25/0.25/0.25/0.15/0.10 | 全局聚合权重 |
| Q5 首次占位 | 0.5 | 等待下次同类题验证 |

#### 定稿确认

- 理论来源自洽（Flavell 元认知理论）✅
- 独占数据源原则贯彻 ✅
- 更新逻辑闭环 ✅
- 与其他维度边界清晰 ✅

### S — 执行稳定

**本质**：学生实际表现与真实能力之间的偏差。不是"会不会"（那是K/A/T），而是"稳不稳"。

**通俗说法**："明明会为什么做错了？"

**核心属性**：
- **全局的**，一个学生一个S
- **不是能力指数**，不搞"增长+上限"那套
- **是失常模式的统计归档**，服务于诊断和干预

#### 七个子维度

| 编号 | 子维度 | 含义 | 数据来源 | 性质 | 是否计入S_global |
|------|--------|------|---------|------|----------------|
| S0 | 波动指数 | 预期得分率与实际得分率的绝对偏差 | K/A/T综合计算 | 客观 | ✅ 是 |
| S1 | 计算稳定性 | 计算过程是否可靠 | 思维链回溯 | 主观 | ✅ 是 |
| S2 | 时间管理 | 时间压力下是否稳定 | 思维链回溯 | 主观 | ✅ 是 |
| S3 | 审题稳定性 | 审题是否仔细 | 思维链回溯 | 主观 | ✅ 是 |
| S4 | 步骤稳定性 | 步骤执行是否流畅 | 思维链回溯 | 主观 | ✅ 是 |
| S5 | 状态稳定性 | 做题前的生理/心理状态 | 回溯结束时轻量自评 | 主观（归因） | ❌ 否 |
| S6 | 考场心态 | 考试中的心理波动 | 考试级上传时询问 | 主观（归因） | ❌ 否 |

#### S0 波动指数算法

**单次偏差 d**：
```
P_exp = w_K × K_mastery + w_A × A_unit + w_T × T_global

单元内题：w_K=0.3, w_A=0.7, w_T=0
跨单元题：w_K=0.2, w_A=0.4, w_T=0.4

R_actual = 学生实际得分 / 该题满分
d = |P_exp - R_actual|
```

**波动指数 V** = 最近N次d的平均值（N=20~30）

**各维度波动**：
```
V_K = 最近N次d_K的平均值（仅当涉及K偏差时）
V_A = 最近N次d_A的平均值（仅当涉及A偏差时）
V_T = 最近N次d_T的平均值（仅当涉及T偏差时）
```
d_K, d_A, d_T 由LLM从错因分析中拆解。

#### 全局稳定性指数 S_global

**公式**：
```
S_global = w_self × Self_stability + w_obj × Obj_stability

Self_stability = 1 - Σ(w_i × 失常率_i)   （自报部分，来自S1~S4）
Obj_stability = 1 - V                     （客观部分，来自S0波动指数）

w_self = 0.6, w_obj = 0.4
```

**Self_stability 详细计算**：
```
失常率_i = S_i的失常次数 / S_i的总观测次数（i=1~4）
w_i（动态权重）= 失常率_i / Σ(失常率_j)
```
失常率越高的子维度权重越大，S_global 重点反映最薄弱的稳定性环节。

**Obj_stability 详细计算**：
```
V = 最近N次d的平均值（N=20~30）
Obj_stability = 1 - V
```

**计算举例**：
```
学生数据：S1失常率=0.35, S2=0.05, S3=0.12, S4=0.08, V=0.18

Step 1 — Self_stability：
  Σ失常率 = 0.35+0.05+0.12+0.08 = 0.60
  w_i:  S1=0.35/0.60=0.583,  S2=0.05/0.60=0.083
        S3=0.12/0.60=0.200,  S4=0.08/0.60=0.133
  Σ(w_i×失常率_i) = 0.583×0.35 + 0.083×0.05 + 0.200×0.12 + 0.133×0.08
                  = 0.204+0.004+0.024+0.011 = 0.243
  Self_stability = 1-0.243 = 0.757

Step 2 — Obj_stability：
  Obj_stability = 1-0.18 = 0.820

Step 3 — S_global：
  S_global = 0.6×0.757 + 0.4×0.820 = 0.454+0.328 = 0.782
```

#### S 与 K/A/T 的边界

| 情况 | 归属 |
|------|------|
| K/A/T达标，做错了 | S问题（失常） |
| K/A/T达标，做对了但步骤异常 | S问题（隐性损耗） |
| K/A/T不达标，做错了 | K/A/T问题，不进S |
| K/A/T不达标，做对了（侥幸） | K/A/T问题，不进S |
| 预期低但实际高（超常） | S0计入波动，但不视为失常 |

#### 数据来源

S0：K/A/T 综合计算（自动，全量）
S1~S4：思维链回溯（手动，稀疏）
S5~S6：轻量自评（手动，稀疏）

## 四、各维度数据源独立性（非重叠原则）

### 核心原则

**每个维度有且只有自己独占的信号源，一个信号不能同时用于计算两个维度。**

### 当前确认的信号分配

| 维度 | 独占信号 | 不使用的信号 |
|------|---------|------------|
| K | 难度×过程分 | 无（加权得分法0外部参数） |
| A | D×η（有效刺激量），A/U框架 | 不直接使用对/错结果 |
| T | W×D×r（跨类型×鉴别力×解题质量），T/U框架 | 不处理单点对错 |
| Q | 思维链文本（弹窗触发，独占5维评分） | 不接受任何代理指标 |
| S | S0波动指数（客观）+ S1~S4失常自报（主观） | 不判断"这是粗心还是不会" |

### 跨维度参数传递

K 维度的加权得分法不需要外部参数。

```
K 的计算只依赖于：
  难度 0.01-0.99 ← LLM 判定（5维度加权）
  过程分 0-1    ← LLM 判定（4档）
  
  不需要 P(S)、P(G)、P(L₀)、P(T) 等参数
```

K 向其他维度传递的信号：
```
- 具体到题目的过程分和思维链 → 提供给 Q 维度
- 超纲/不超纲标记 → 提供给学习进度指标
```

```
❌ K 的遗忘衰减依赖 Q 维度的理解标记 → 但这是协同关系，不是信号重叠
❌ Q 在掌握度序列上做回归，K 也在同一序列上算掌握度 → 两个维度用同一数据
✅ 解决方案：Q 只看思维链，K 只看对错+难度+过程分，互不干扰
```

## 五、认知科学依据

| 维度 | 唯一数据 | 不依赖 | 覆盖率 |
|------|---------|--------|--------|
| K | 难度×过程分 | 无（无外部参数） | 全量 |
| A | D×η（有效刺激量），A/U框架 | LLM对η的判定准确性 | 全量 |

| Q | 思维链文本 | 学生写思维链的意愿 | 稀疏 |

**五维不重叠**：每个维度有自己独占的信号源，不存在两个维度用同一数据算不同分的情况。
