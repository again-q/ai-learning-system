# 五维能力模型 — 学科掌握诊断框架

> 理论构建：again-q
> 版本：v2.5（2026-07-13）  
> 状态：K/A/Q 已定稿，T 待定（依赖知识图谱），S 逻辑已通  
> 定位：**课后辅助诊断工具，不代替学校**

---

## 一、核心命题

用一个可量化的、可操作的模型，让一个初高中生知道自己"到底哪里不会"、"为什么不会"、"下一步该做什么"。

三个子问题：
1. **"掌握"到底指什么？**—— 不只是"会不会"，而是"会到什么程度"
2. **怎么把它变成可测的数字？** —— 从学生日常做题数据中提取五个维度的信号
3. **测出来之后怎么用？** —— 输出可操作的诊断报告和行动建议

---

## 二、核心 Pipeline

所有维度的计算基于统一的数据管道：

```
拍照 → AI 识别题目 + 学生作答 → 对比正确答案
    ↓
LLM 错因分类（将"错"拆为 4 类）
    ↓
计算失误/抄错数字 → S 维度扣分
套错公式但结构对   → A 维度扣分
题意理解偏/答非所问 → T/K 维度扣分
完全空白/乱填      → K 维度扣分
    ↓
    高价值题目？ → 弹窗问思维链 → Q 评分
    ↓
更新五维分数 → 输出诊断报告
```

**核心原则**：每个维度使用互不重叠的数据源，避免维度间的信号污染。

---

## 三、五维能力向量

```
M = (K, A, T, Q, S)
```

| 维度 | 对学生说 | 认知本质 | 粒度 | 数据源 | 覆盖率 |
|------|---------|---------|------|--------|--------|
| K | 知不知道？ | 陈述性记忆 | 知识点 | 难度×过程分（加权得分法） | 全量 |
| A | 会不会用？ | 程序性知识流畅度 | 教材单元 | D×η（有效刺激量），A/U框架 | 全量 |
| T | 能不能串？ | 跨单元结构识别 | 跨单元 | 知识图谱DAG（待实现） | 依赖图谱 |
| Q | 会不会学？ | 元认知（觉察与调整） | 全局 | 思维链（弹窗触发，独占） | 稀疏 |
| S | 稳不稳？ | 预期vs实际偏差 | 全局 | K/A/T综合预期对比 | 全量 |

---

### K — 知识掌握

**本质**：知识点在你的长时记忆里存好了吗？

**通俗说法**："知不知道"

**三层结构**：
| 层 | 内容 | 计入K总分 |
|---|------|---------|
| 知识层 | 概念定义、教材方法 | ✅ 是 |
| 模型层 | 题目模型、解题框架 | ❌ 参考 |
| 方法层 | 思维技巧、解题策略 | ❌ 参考 |

**算法**：加权得分法（S_k / D_k）

**基本公式**：
```
每道题贡献 = 难度(0.01-0.99) × 过程分(0/0.3/0.5/1.0)
知识点累计得分 S_k = Σ 贡献
知识点累计难度 D_k = Σ 难度
掌握度 = S_k / D_k
```

**难度判定（LLM，5维度加权平均）**：
| 维度 | 低分 | 高分 |
|------|------|------|
| 知识复杂度 | 单步套公式 | 多步+跨知识点 |
| 思维深度 | 直接套用 | 逆向/构造 |
| 陷阱密度 | 直白无坑 | 多陷阱/分类讨论 |
| 计算强度 | 口算级别 | 复杂运算 |
| 陌生度 | 常规题 | 新题型 |

**过程分判定（LLM）**：
- 1.0 → 完全正确，思路清晰
- 0.5 → 部分正确 / 正确但思路模糊
- 0.3 → 错误但有正确思路
- 0 → 完全错误 / 空白

**超纲规则**：LLM 判定超纲的题目，做错时不降掌握度（不计入 D_k）
**遗忘机制**：超过 14 天未练习触发，有 Q 维度理解标记的衰减减半
**话题级聚合**：
- 话题（如"二次函数"）下各知识点独立评分
- 话题整体掌握度 = 各知识点掌握度的加权平均
- 同步展示该话题下最薄弱的 1-3 个知识点

**展示**：4 档（掌握/巩固/薄弱/紧急） + 学习进度指标（待实现）

**K 维度完整决策记录**：
| 决策点 | 结论 |
|--------|------|
| 底层算法 | 加权得分法（非 BKT），S_k/D_k |
| 过程分 | LLM 4 档（1.0/0.5/0.3/0） |
| 难度 | LLM 5 维度判定，0.01-0.99 连续值 |
| 超纲 | 超纲做错不计入 D_k |
| 遗忘 | 14 天触发，有 Q 理解标记衰减减半 |
| P(T) 学习率 | 不使用（只诊断不教学） |
| 话题级聚合 | 加权平均 + 展示最薄弱 |
| 模型层/方法层 | 参考展示，不参与评分 |
| 4 档边界 | 等有数据再定

**七要素归属**：
| 要素 | 归属维度 |
|------|---------|
| 定义、性质、操作、关系、表示 | K 维度 |
| 应用 | A 维度 |
| 来源 | Q 维度 |

### A — 应用能力

**本质**：运用所学知识解题的能力指数。不是"做对的概率"，而是运用知识解决问题的水平高低。

**通俗说法**："会不会用"

**粒度**：教材单元（如"牛顿第二定律"一个单元维护一个 A）

**核心区别**：
```
K = 知不知道这个公式（陈述性记忆）
A = 能不能识别出该用这个公式并正确运用（程序性知识流畅度）
T = 能不能把多个公式串起来（跨单元串联）
```

**核心思想**：
1. A 是能力指数，不是概率 — 随着有效练习增长，不会围绕固定值波动
2. 学习是分单元进行的 — 每个单元独立维护自己的 A 和 U
3. A 的增长受上限 U 约束 — A 不能超过 U，U 本身由解题质量决定
4. 路径质量 η 决定增速和上限走向

**两层结构**：

| 粒度 | 名称 | 数量 | 含义 |
|------|------|------|------|
| 单元级 | 单元A | 每单元一个 | 该单元内的应用能力指数 |
| 全局级 | 全局A | 每学生一个 | 所有单元A加权聚合，反映整体应用能力 |

### 单元A核心算法

**每个单元维护的数据**：

| 变量 | 含义 | 初始值 | 范围 |
|------|------|--------|------|
| A | 当前能力指数 | 0.30 | [0, U] |
| U | 当前能力上限 | 0.50 | [A, 1] |
| N | 总观测次数 | 0 | 整数 |

**有效刺激量 E**：
```
E = D × η
D = 题目鉴别力（LLM复用K维度5维度，权重调整，0~1）
η = 路径质量系数（LLM分析步骤文本，判断解法是否接近本质，0.4~1.0）

做对时：E > 0（正向刺激）
做错且是A问题：E < 0（负向刺激，取 -D×η）
做错且是K/S问题：E = 0（不更新A）
```

**上限U的更新**：
```
if E >= 0.5 and η >= 0.7:
    # 上浮：学生展示了本质解法，有突破潜力
    U += 0.05 × (1 - U)
elif 连续5道题 η <= 0.6:
    # 下浮：学生持续暴力计算，挤压虚假上限
    U -= 0.03 × (U - A)
```

**能力指数A的更新**：
```
if E > 0:
    ΔA = 0.25 × E × (U - A)
    A = min(A + ΔA, U)       # 硬约束：A不超过U
elif E < 0:
    ΔA = 0.25 × |E| × (U - A)
    A = max(A - ΔA, 0)       # 不低于0
```

**展示**：两位小数（百分位）

### 全局A

全局A = 各单元A加权聚合，经EWMA平滑。

**权重公式**：
```
w_i = importance_i × confidence_i × stage_i
importance_i = 大纲内置（静态，反映高考占比）
confidence_i = 1 - e^{-0.1 × N_i}（动态，数据量越大越可信）
stage_i = 0.7 + 1.0 × active_i（动态，正在学的单元权重更高）
```

**平滑**：EWMA，α_g = 0.25

**展示条件**：有效单元数 ≥ 2 且 总N ≥ 15

**数据来源**：D（题目鉴别力，LLM复用K维度）、η（路径质量，LLM分析步骤文本）、对/错+错因（区分A/K/S问题）

### T — 迁移能力

**本质**：不同知识点之间的触类旁通——把不同知识串起来解决复杂问题

**通俗说法**："触类旁通"

**递进关系**：
```
K = 一个知识点"知不知道"
A = 一个知识点"会不会用"（同知识点不同场景）
T = 多个知识点"能不能串起来"（跨知识点聚合）
```

**算法**：基于知识图谱 DAG 的跨节点表现
- 正迁移：连续做对两个关联知识点 → T 上升
- 负迁移：错误地将 B 知识点公式用到 A 知识点 → T 下降

**前提**：依赖学科知识图谱的搭建

---

### Q — 思维品质

**本质**：觉察和调整的能力 —— 你知不知道你怎么错的，下次能不能改过来

**通俗说法**："会不会学"

**核心原则**：
- Q 只通过思维链测量。没有思维链，Q 不更新。
- 思维链不让学生主动写，而是 AI 判断为高价值题目后弹窗询问
- 对题也能涨 Q（思路清晰说明觉察能力强）

**触发方式**：AI 识别高价值题目 → 弹窗问"你是怎么想的？"
- 高价值判断：做错的题、做对但过程复杂的题、涉及关键知识节点的题
- Q 低 → 每次高价值题都弹窗
- Q 高 → 减弹窗，鼓励主动写

**评分方式（有思维链时）**（✅ 已定稿）：
```
LLM 评思维链的三个维度：
① 归因准确度 — 是否准确说出了错因/思路？
② 调整清晰度 — 是否说了下次怎么改？
③ 执行结果   — 下次同类题是否做对了？

对题：思路清晰 → Q +
错题：归因准 + 调整明确 → Q ++
无思维链 → Q 不更新（维持原值或微幅衰减）
```

**理论依据**：Flavell 元认知理论（监测、控制、评估）。思维链是 Q 的唯一可观测窗口。自我觉察是内在心理过程，只能通过外在的语言表达（思维链）来推断。

**定稿确认**（2026-07-13）：
- 理论来源自洽 ✅
- 独占数据源原则贯彻 ✅
- 更新逻辑闭环 ✅
- 与其他维度边界清晰 ✅
- 交互设计（弹窗频率/冷启动/质量过滤）为工程问题，不影响理论自洽性

---

### S — 执行稳定

**本质**：预期偏差 —— 你实际发挥相对于你应该发挥水平的偏离程度

**通俗说法**："稳不稳定"、"明明会为什么做错了？"

**认知本质**：预期 vs 实际的偏差
- 低于预期（失常）→ S 下降
- 高于预期（超常）→ S 微涨（权重更低，因为可能是偶然）

**算法**：
```
预期值   = 由 K/A/T 综合算出的"这道题你应该做对的概率"
单次偏差 = 实际结果(1/0) - 预期概率
S       = 过去 N 次偏差的加权移动平均
          （失常权重 1.0，超常权重 0.3）
```

**数据来源**：K/A/T 综合预期 vs 实际对错结果（全量）

**与错因分类的关系**：计算失误/抄错数字是 S 维度的主要负向信号，但不是唯一信号。

## 四、各维度数据源独立性（非重叠原则）

### 核心原则

**每个维度有且只有自己独占的信号源，一个信号不能同时用于计算两个维度。**

### 当前确认的信号分配

| 维度 | 独占信号 | 不使用的信号 |
|------|---------|------------|
| K | 难度×过程分 | 无（加权得分法0外部参数） |
| A | D×η（有效刺激量），A/U框架 | 不直接使用对/错结果 |
| T | 跨知识点关联作答模式（待实现） | 不单独处理单点对错 |
| Q | 思维链文本（弹窗触发，独占） | 不接受任何代理指标 |
| S | 实际表现 vs K/A/T 预期偏差 | 不自行判断"这是粗心还是不会" |

### 跨维度参数传递

K 维度的加权得分法不需要外部参数。

```
K 的计算只依赖于：
  难度 0.01-0.99 ← LLM 判定（5维度加权）
  过程分 0-1    ← LLM 判定（4档）
  
  不需要 P(S)、P(G)、P(L₀)、P(T) 等参数
```

K 向其他维度传递的信号：
```
- 具体到题目的过程分和思维链 → 提供给 Q 维度
- 超纲/不超纲标记 → 提供给学习进度指标
```

```
❌ K 的遗忘衰减依赖 Q 维度的理解标记 → 但这是协同关系，不是信号重叠
❌ Q 在掌握度序列上做回归，K 也在同一序列上算掌握度 → 两个维度用同一数据
✅ 解决方案：Q 只看思维链，K 只看对错+难度+过程分，互不干扰
```

## 五、认知科学依据

| 维度 | 唯一数据 | 不依赖 | 覆盖率 |
|------|---------|--------|--------|
| K | 难度×过程分 | 无（无外部参数） | 全量 |
| A | D×η（有效刺激量），A/U框架 | LLM对η的判定准确性 | 全量 |
| T | 知识图谱跨节点关联 | 图谱搭建完成度 | 条件 |
| Q | 思维链文本 | 学生写思维链的意愿 | 稀疏 |
| S | K/A/T 综合预期 vs 实际 | 其他维度的准确度 | 全量 |

**五维不重叠**：每个维度有自己独占的信号源，不存在两个维度用同一数据算不同分的情况。

## 七、适配层次

| 层次 | 学段 | 推荐维度 | 重心 |
|------|------|---------|------|
| 筑基期 | 高一 | K 主导 | 扫清知识盲区 |
| 连接期 | 高二 | K + A + T | 建立知识网络 |
| 兑现期 | 高三 | A + S 主导 | 稳定发挥能力 |

**核心观点**：同一个学生不同阶段的重心不同，五维模型动态适配。

## 八、一句话总结

> 一个拍照上传题目就能出诊断报告的学习分析工具。不搜题、不讲课、不刷题——只做一件事：**告诉你到底哪里不会，为什么不会，然后怎么学。**
